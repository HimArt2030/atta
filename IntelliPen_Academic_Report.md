# مشروع التخرج: IntelliPen - إطار اختبار اختراق ذكي مؤتمت بالكامل

**إعداد:** طالب الأمن السيبراني الهجومي
**إشراف:** دكتور المقرر
**التاريخ:** أبريل 2026

---

## 1. الملخص التنفيذي (Executive Summary)

يُعد مشروع **IntelliPen** نقلة نوعية من مجرد "أدوات تعليمية" إلى **سلاح سيبراني هجومي متكامل وواقعي**. تم تصميم هذا الإطار لمعالجة الفجوة بين الفحص الآلي التقليدي والتحليل البشري المعقد، من خلال دمج محركات استطلاع وفحص متقدمة مع **محرك ذكاء اصطناعي محلي (Local AI Exploitation Engine)**. 

لا يكتفي النظام باكتشاف الثغرات فحسب، بل يقوم بتحليلها، وتقييم مستوى خطورتها، وتوليد حمولات (Payloads) مخصصة لاستغلالها، وبناء سلاسل هجوم (Attack Chains) كاملة، تماماً كما يفعل مختبر الاختراق المحترف.

---

## 2. المشكلة والحل (Problem & Solution)

### 2.1 المشكلة في الأدوات الحالية
معظم مشاريع التخرج في الأمن السيبراني تقتصر على بناء أدوات تعليمية تشرح المفاهيم، أو أدوات فحص تقليدية (Scanners) تعتمد على التواقيع الثابتة (Signature-based) وتنتج الكثير من الإيجابيات الخاطئة (False Positives) دون تقديم استراتيجية استغلال فعلية.

### 2.2 الحل: IntelliPen
يقدم IntelliPen حلاً هجومياً واقعياً يتكون من:
1. **محرك استطلاع ذكي (Smart Recon):** يجمع معلومات دقيقة عن الهدف (تقنيات، منافذ، ملفات حساسة، نقاط نهاية API).
2. **محرك فحص ديناميكي (Dynamic Scanner):** يكتشف الثغرات المعقدة (SQLi, XSS, IDOR, CSRF, Path Traversal) باستخدام تقنيات حقن متقدمة.
3. **محرك استغلال بالذكاء الاصطناعي (AI Exploitation Engine):** يحلل الثغرات المكتشفة، يولد حمولات مخصصة لتجاوز الحماية (WAF/Filters)، ويرسم مسار الهجوم الأمثل.
4. **واجهة تحكم تفاعلية (Web Dashboard):** تتيح إدارة العمليات وعرض النتائج في الوقت الفعلي.
5. **مولّد تقارير احترافي (PDF Generator):** يصدر تقارير جاهزة للاستخدام المؤسسي.

## 3. البنية المعمارية للنظام (System Architecture)

تم بناء النظام باستخدام بنية معيارية (Modular Architecture) تضمن قابلية التوسع والتطوير:

| المكون (Component) | التقنية المستخدمة (Technology) | الوظيفة (Function) |
|---|---|---|
| **الواجهة الخلفية (Backend)** | Python 3.11, Flask | إدارة الجلسات، توجيه الطلبات، وتنسيق عمل المحركات |
| **الواجهة الأمامية (Frontend)** | HTML5, CSS3, JavaScript, Socket.IO | عرض النتائج في الوقت الفعلي، تفاعل المستخدم |
| **محرك الاستطلاع (Recon)** | Requests, BeautifulSoup4, Socket | جمع المعلومات، تحليل الرؤوس، تعداد الدلائل |
| **محرك الفحص (Scanner)** | Custom Python Payloads | حقن الحمولات، تحليل الاستجابات، اكتشاف الثغرات |
| **محرك الذكاء الاصطناعي (AI)** | Local Knowledge Base, Heuristics | تحليل الثغرات، توليد الحمولات، بناء سلاسل الهجوم |
| **مولّد التقارير (Reporter)** | FPDF | إنشاء تقارير PDF احترافية ومفصلة |

## 4. الميزات الهجومية المتقدمة (Advanced Offensive Features)

### 4.1 الاستطلاع العميق (Deep Reconnaissance)
- **بصمة التقنيات (Technology Fingerprinting):** تحليل رؤوس HTTP ومحتوى HTML لاكتشاف التقنيات المستخدمة (مثل Wappalyzer).
- **تعداد الدلائل (Directory Enumeration):** فحص المسارات المخفية والملفات الحساسة (مثل `.env`, `config.php`, `.git`).
- **استخراج نقاط النهاية (API Endpoint Extraction):** تحليل ملفات JavaScript لاستخراج مسارات API المخفية.

### 4.2 الفحص الذكي للثغرات (Smart Vulnerability Scanning)
- **SQL Injection:** اختبار الحقن القائم على الخطأ (Error-based)، والاتحاد (Union-based)، والزمني (Time-based).
- **Cross-Site Scripting (XSS):** اختبار XSS المنعكس والمخزن باستخدام حمولات تتجاوز الفلاتر البسيطة.
- **Insecure Direct Object Reference (IDOR):** التلاعب بالمعلمات (Parameters) لاكتشاف الوصول غير المصرح به.
- **Security Headers:** تحليل الرؤوس الأمنية المفقودة وتقييم تأثيرها.

### 4.3 محرك الذكاء الاصطناعي المحلي (Local AI Engine)
نظراً للقيود الأمنية والاعتمادية على واجهات برمجة التطبيقات الخارجية (APIs)، تم بناء محرك ذكاء اصطناعي محلي يعتمد على:
- **قاعدة معرفة ضخمة (Knowledge Base):** تحتوي على مئات الحمولات المتقدمة وتقنيات التجاوز.
- **تحليل السياق (Contextual Analysis):** اختيار الحمولة المناسبة بناءً على نوع قاعدة البيانات (MySQL, PostgreSQL, MSSQL) والتقنيات المكتشفة.
- **بناء سلاسل الهجوم (Attack Chaining):** ربط الثغرات ببعضها (مثلاً: SQLi -> استخراج بيانات الاعتماد -> IDOR -> RCE).

## 5. إثبات المفهوم والاختبار العملي (Proof of Concept & Testing)

لإثبات فعالية النظام بشكل واقعي، تم بناء **بيئة اختبار محلية (Vulnerable Target Lab)** تحتوي على تطبيق ويب مصاب بثغرات متعمدة ومعقدة.

### 5.1 بيئة الاختبار (Target Lab)
- تطبيق مبني بـ Flask و SQLite.
- يحتوي على ثغرات: SQLi (في تسجيل الدخول)، XSS (في البحث)، IDOR (في الملف الشخصي)، CSRF (في التحويل المالي)، و Command Injection.

### 5.2 نتائج الفحص (Scan Results)
عند توجيه IntelliPen نحو بيئة الاختبار، أظهر النتائج التالية في أقل من دقيقتين:
- **الاستطلاع:** اكتشاف التقنيات (Python/Flask)، المنافذ المفتوحة، و 4 مسارات حساسة (`/admin`, `/api/v1/users`).
- **الثغرات المكتشفة:** 30 ثغرة مؤكدة، منها 17 ثغرة عالية الخطورة (High) و 7 متوسطة (Medium).
- **تحليل الذكاء الاصطناعي:** حدد مستوى الخطر بـ **MEDIUM/HIGH**، واقترح سلسلة هجوم تبدأ بـ XSS لسرقة الجلسات (Account Takeover).
- **التقرير:** تم توليد تقرير PDF احترافي يحتوي على تفاصيل الثغرات، الحمولات الناجحة، والتوصيات الأمنية.

## 6. التميز الأكاديمي والتقني (Academic & Technical Excellence)

يتجاوز هذا المشروع متطلبات التخرج التقليدية من خلال:
1. **البرمجة الفعلية من الصفر:** لم يتم استخدام أدوات جاهزة (مثل Nmap أو SQLMap)، بل تم بناء المحركات برمجياً بالكامل باستخدام Python.
2. **الواقعية والفعالية:** النظام قادر على فحص مواقع حقيقية واكتشاف ثغرات فعلية (Zero-day potential).
3. **الاستقلالية (Air-gapped Capability):** محرك الذكاء الاصطناعي يعمل محلياً دون الحاجة لاتصال بالإنترنت أو اشتراكات مدفوعة (OpenAI/Claude)، مما يجعله مثالياً للبيئات الحساسة.
4. **الواجهة الاحترافية:** تصميم UI/UX حديث يحاكي المنتجات التجارية (Enterprise-grade).

مشروع **IntelliPen** يمثل تطبيقاً عملياً ومتقدماً لمفاهيم الأمن السيبراني الهجومي. إنه يثبت القدرة على تحويل المعرفة النظرية إلى أداة برمجية قوية وفعالة، قادرة على محاكاة تفكير المهاجم البشري وأتمتة المهام المعقدة، مما يجعله مشروع تخرج استثنائي يرقى لمستوى التميز الأكاديمي.
